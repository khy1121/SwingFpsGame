### 발생한 문제점 및 해결 과정


**서버-클라이언트 오브젝트 동기화 오류**
  - 현상: 오브젝트 ID, HP, 팀 정보가 불일치하거나 파괴 후 잔상 남는 현상 발생
  - 해결 과정: 서버에서 오브젝트 상태 변경 시마다 OBJ_UPDATE/OBJ_DESTROY 메시지를 모든 클라이언트에 브로드캐스트하도록 수정. 클라이언트는 서버의 오브젝트 목록을 주기적으로 동기화하며, 파괴된 오브젝트는 즉시 제거하도록 로직을 보강. 테스트를 통해 동기화 지연 및 누락 케이스를 반복 검증하여 정상 동작 확인.

**지뢰/터렛 판정 버그**
  - 현상: 지뢰가 밟혀도 폭발하지 않거나, 터렛이 적을 자동공격하지 않는 문제
  - 해결 과정: 서버에서 오브젝트별 판정 범위와 주기를 명확히 정의하고, 일정 주기로 모든 플레이어 위치를 검사하여 조건 충족 시 폭발/공격 이벤트를 발생시키도록 TimerTask를 활용. 판정 결과를 PLACE/OBJ_UPDATE/TURRET_SHOOT 등 프로토콜로 클라이언트에 즉시 전송. 여러 상황에서 반복 테스트하여 정상 폭발/공격 동작을 확인.

**미사일-오브젝트 충돌 판정 누락**
  - 현상: 미사일이 오브젝트에 명중해도 서버에 피격 보고가 누락되는 경우
  - 해결 과정: 클라이언트에서 미사일이 오브젝트에 충돌 시 HIT_OBJ 메시지를 서버로 전송하도록 구현. 서버는 해당 메시지를 수신하면 오브젝트 HP를 감소시키고, 파괴 시 OBJ_DESTROY를 브로드캐스트. 충돌 판정의 신뢰성을 위해 서버-클라이언트 양방향 검증 로직을 추가.

**에어스트라이크 타겟/임팩트 시각화 미동작**
  - 현상: 미니맵/메인맵에 마커, 폭발 효과가 제대로 표시되지 않음
  - 해결 과정: 서버에서 STRIKE_MARK(타겟 지정), STRIKE_IMPACT(폭발 발생) 메시지를 명확히 구분하여 전송. 클라이언트는 각 메시지 수신 시 미니맵/메인맵에 마커 및 폭발 이펙트를 즉시 표시하도록 SkillEffectManager와 GamePanel을 수정. 실제 플레이 상황에서 정상 표시되는지 반복 확인.

**버프/디버프 적용/해제 타이밍 불일치**
  - 현상: 오라 범위 진입/이탈 시 버프가 즉시 반영되지 않거나 해제 누락
  - 해결 과정: 서버에서 모든 플레이어의 위치를 주기적으로 검사하여 오라 범위 내/외 여부를 판정, BUFF/UNBUFF 메시지를 실시간으로 전송. 클라이언트는 해당 메시지 수신 즉시 능력치/이펙트 반영. 경계 상황(진입/이탈 반복)에서의 타이밍 문제를 집중 테스트하여 개선.

**UI/UX 혼란 및 정보 부족**
  - 현상: 오브젝트 HP, 소유자, 팀 정보 등 시각적 피드백 부족
  - 해결 과정: GamePanel 및 미니맵에 오브젝트 HP바, 소유자/팀 라벨, 경고 이펙트 등을 추가 구현. SkillEffectManager를 활용해 각종 효과를 시각적으로 명확히 표현. 플레이어 피드백을 반영해 UI/UX를 반복 개선.

**불필요/미구현 코드로 인한 컴파일 경고**
  - 현상: 미사용 효과 클래스, 미구현 캐릭터 스킬 등으로 인한 경고 다수
  - 해결 과정: CharacterData, SkillEffectManager 등에서 미구현/미사용 클래스를 정리 및 삭제. 컴파일 경고가 모두 사라지는지 확인하며, 필요 없는 코드가 남지 않도록 주기적으로 점검.

**멀티 클라이언트 환경에서의 동기화 지연**
  - 현상: 여러 클라이언트가 동시에 오브젝트/스킬을 사용할 때 상태 불일치 발생
  - 해결 과정: 서버가 모든 오브젝트/스킬 상태의 최종 권위를 갖도록 설계. 클라이언트는 서버의 브로드캐스트 메시지에 따라 상태를 즉시 갱신. 동시 접속 환경에서 반복 테스트하며, 지연/불일치 현상 발생 시 서버 로직을 추가 보강.

**테스트 중 서버 포트 충돌/중복 실행**
  - 현상: 서버 재시작 시 포트가 해제되지 않아 실행 불가
  - 해결 과정: 서버 종료 시 소켓 및 리소스를 명확히 해제하도록 try-catch-finally 구조로 보강. 예외 발생 시 사용자에게 안내 메시지 출력 및 재시작 대기 로직 추가. 여러 번 재시작/종료를 반복하며 포트 충돌이 재발하지 않는지 확인.

### 문제 해결 과정에서 배운 점

- **실시간 동기화의 중요성**: 서버가 모든 오브젝트/스킬의 최종 상태를 관리하고, 클라이언트는 서버의 브로드캐스트 메시지에 따라 상태를 즉시 갱신해야 일관성이 유지됨을 체감함. 동기화 지연이나 누락이 게임 플레이에 직접적인 영향을 미치므로, 주기적 동기화와 즉각적인 상태 반영이 필수적임을 배움.

- **서버 권위 기반 설계의 필요성**: 멀티 클라이언트 환경에서 판정 및 상태 관리의 권한을 서버에 집중시켜야 예측 불가능한 동기화 오류와 상태 불일치를 방지할 수 있음을 확인함.

- **프로토콜 설계와 메시지 명확성**: 다양한 이벤트(폭발, 버프, 오브젝트 파괴 등)를 명확히 구분하는 프로토콜 메시지 설계가 중요함을 실감. 메시지의 의미와 처리 시점을 명확히 해야 클라이언트와 서버가 오동작 없이 동작함.

- **UI/UX와 시각적 피드백의 효과**: HP바, 라벨, 이펙트 등 시각적 정보 제공이 플레이어의 몰입도와 게임 이해도를 크게 높임을 경험함. 반복적인 피드백 반영과 개선이 중요함을 느낌.

- **코드 정리와 유지보수의 중요성**: 미사용/미구현 코드를 주기적으로 정리함으로써 컴파일 경고와 예기치 않은 버그를 예방할 수 있음을 확인. 코드의 간결성과 명확성이 유지보수에 큰 도움이 됨.

- **예외 처리와 안정성 확보**: 서버 종료/재시작, 네트워크 예외 등 다양한 상황에서의 예외 처리가 실제 운영 환경에서 매우 중요함을 체감. try-catch-finally 구조와 사용자 안내 메시지의 필요성을 배움.

- **반복 테스트와 검증의 필요성**: 모든 기능/버그 수정 후 다양한 시나리오에서 반복 테스트를 수행해야 예상치 못한 문제를 조기에 발견하고, 신뢰성 있는 결과를 얻을 수 있음을 경험함.
